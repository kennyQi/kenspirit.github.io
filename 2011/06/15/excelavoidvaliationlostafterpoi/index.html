如何避免Microsoft Excel在Cell上定义好的Named Validation的下拉选项在POI写值后丢失

By 鹄思乱想

Jun 15 2011 Updated:Mar 28 2014

Contents

有一种常用的 Excel 报表打印实现是用客户定义好的 Excel Template, 通过 POI 来写值进去。

这样的好处是基本可以原汁原味的保留 Excel 的样式和一些宏。

有时，在 Excel 里会定义有一些 Name, 然后用到某一些 Cell 里做 Validation, 也就是模拟下拉选项的效果。

有时候，Excel Template 经过 POI 写值后，会发现那个用 Validation 做成的模拟下拉选项不见了，而未写值前的 Excel Template 里是有的。 情况可能是这样：

  1. 在做好 Excel Template, 保存的瞬间，光标选在的某个有 Named List Validation 的 Cell 上。我们要注意，保存 Excel Template 里，光标必须落在一些空白或不重要的地方。

  2. 如果你做的 Named List 里动态写值的，那最好先做一些 dummy 的 list, 在需要做 Validation 的 Cell 上，定义好这个 Named List Validation, 然后再把那些 dummy value 删去。

这样都会有影响，Microsoft 的东西有够诡异的了吧？ 哈哈
